# Paper Judging System (Sistem Penilaian Makalah)

Aplikasi web modern berbasis **Next.js (App Router)** yang dirancang untuk mempermudah proses penilaian dan rekapitulasi skor makalah secara digital dan real-time. 

Sistem ini memisahkan akses antara juri publik dengan Admin untuk menjaga validitas data, dilengkapi dengan fitur Leaderboard (Podium) otomatis.

## Fitur & Alur Halaman (Routing)

Aplikasi ini menggunakan struktur Next.js App Router dengan pembagian halaman sebagai berikut:
* 🏠 `/` : Halaman Utama / Landing Page.
* 🔐 `/login` : Sistem autentikasi untuk masuk ke mode penilaian atau dashboard admin.
* 📝 `/nilai/[id]` : Halaman dinamis bagi juri/user untuk memberikan skor pada makalah spesifik.
* 📊 `/admin` : Dashboard utama untuk memantau seluruh data makalah dan total voting.
* 🔍 `/admin/detail/[id]` : Halaman detail untuk melihat rincian skor dari satu makalah.
* 🥇 `/admin/podium` : Halaman Leaderboard real-time yang menampilkan ranking makalah dengan nilai tertinggi.

## Tech Stack & Tools

Project ini dibangun dengan teknologi web terbaru untuk menjamin kecepatan (performance) dan keamanan:
* **Framework:** Next.js 16 (App Router)
* **Library:** React 19 + TypeScript
* **Styling:** Tailwind CSS v4
* **Database & Auth:** Supabase (PostgreSQL)
* **Icons & UI:** Phosphor Icons & SweetAlert2 (untuk notifikasi interaktif)

## Cara Menjalankan Project di Localhost

Jika ingin melakukan clone dan mencoba aplikasi ini di perangkat Anda, ikuti langkah-langkah berikut:

### 1. Clone repository ini:
```bash
git clone https://github.com/wyzsee/paper-judging.git
```

### 2. Masuk ke direktori dan install dependencies:
```bash
cd paper-judging
npm install
```

### 3. Setup Environment Variables:
Buat file bernama `.env.local` di root folder, lalu masukkan kredensial Supabase Anda. (Catatan: File ini sudah diabaikan oleh .gitignore demi keamanan).
```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 4. Jalankan development server:
```bash
npm run dev
```
Buka http://localhost:3000 di browser untuk melihat hasilnya.

## Setup Database (Supabase)

Aplikasi ini membutuhkan 3 tabel utama di Supabase (`profiles`, `participants`, dan `scores`) beserta integrasi ke sistem Authentication bawaan Supabase (`auth.users`).

### 1. Buat Tabel (Run SQL Editor)
Masuk ke menu **SQL Editor** di dashboard Supabase project Anda, lalu jalankan query berikut untuk membuat tabel dan relasinya secara otomatis:

```sql
-- 1. Tabel Profiles (Berelasi dengan auth.users bawaan Supabase)
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,
  username TEXT,
  role TEXT,
  full_name TEXT,
  email TEXT
);

-- 2. Tabel Participants (Data Makalah/Peserta)
CREATE TABLE participants (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT,
  moderator TEXT,
  presenters TEXT[],
  image_url TEXT,
  display_order INTEGER
);

-- 3. Tabel Scores (Data Penilaian dari Juri)
CREATE TABLE scores (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  participant_id BIGINT REFERENCES participants(id) ON DELETE CASCADE,
  judge_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  originality INTEGER,
  applicable INTEGER,
  Benefit INTEGER,
  Collaborative INTEGER,
  presentation INTEGER,
  total_score INTEGER
);
```

### 2. Cara Membuat Akun Juri (Untuk Login)
Karena project ini menggunakan Supabase Auth, ikuti langkah berikut untuk membuat akun juri:

1. Buka dashboard Supabase, masuk ke menu **Authentication** -> **Users**.
2. Klik **Add User** -> **Create New User**.
3. Masukkan Email dan Password untuk juri (misal: `juri1@gmail.com`).
4. Centang **"Auto Confirm User"** agar juri bisa langsung login tanpa verifikasi email.
5. **PENTING:** Setelah user terbuat, copy `User UID` dari user tersebut.
6. Masuk ke menu **Table Editor** -> pilih tabel `profiles`.
7. Tambahkan baris baru (*Insert Row*), paste `User UID` tadi ke kolom `id`, lalu isi data `username`, `role` (misal: 'juri'), `full_name`, dan `email` secara manual. 

Selesai! Sekarang juri bisa login menggunakan email dan password yang dibuat pada langkah 3.
